# CRXSM Public Licensing API Contract (`/Tunnel`)

This document outlines the public API endpoints exposed by the CRXSM platform at `/Tunnel/index.php`. All external clients (WordPress plugins, SaaS products, desktop tools) communicate with CRXSM via these endpoints.

---

## 1. Authentication & Integrity

Every HTTP request to the API must include four custom security headers. Any request failing these checks is rejected immediately with a `400 Bad Request` or `401 Unauthorized` status.

### Required HTTP Headers

| Header Name | Description | Example |
| :--- | :--- | :--- |
| `X-CRXSM-Client-ID` | The software-specific API Client ID | `sw_64b2a8d1e39a5` |
| `X-CRXSM-Signature` | Hex-encoded HMAC-SHA256 request signature | `a7d86f...e3b2f1` |
| `X-CRXSM-Timestamp` | Current Unix timestamp (UTC) of the client machine | `1783935293` |
| `X-CRXSM-Nonce` | Unique single-use random string (minimum 16 chars) | `Kj82NsLp02Mxb19s` |

### HMAC Signature Construction

The signature is computed over the payload concatenated with the timestamp and nonce using the software product's Client Secret.

```
data_to_sign = X-CRXSM-Timestamp + "." + X-CRXSM-Nonce + "." + HTTP_RAW_POST_DATA
signature = HMAC-SHA256(data_to_sign, Client_Secret)
```

- **Replay Protection:** Nonces are saved in the CRXSM database. Reusing a nonce within a 5-minute window is rejected.
- **Freshness Window:** If the absolute difference between `X-CRXSM-Timestamp` and the server's current timestamp exceeds 300 seconds (5 minutes), the request is rejected.

## 2. Asymmetric Offline Sanity Verification

License keys generated by CRXSM are structured tokens:
`base64url(payload_json) . "." . base64url(signature)`

The signature is generated using the product's private Ed25519 key on the server. Clients can perform a fast, offline verification using the corresponding public key bundled within the client code:

```php
// PHP Client Example
$parts = explode('.', $license_key);
$payload_json = base64url_decode($parts[0]);
$signature = base64url_decode($parts[1]);

$is_authentic = sodium_crypto_sign_verify_detached($signature, $payload_json, $public_key);
```

> [!WARNING]
> **Offline Validation is Sanity-Only**
> Cryptographic signature verification only confirms the token was issued by CRXSM and has not been tampered with. It **cannot** detect real-time revocation, expiry checks triggered after issuance, or activation count changes. Authoritative status checks require querying the `/Tunnel` endpoints.

## 3. Endpoints Registry

All endpoints are accessed via POST requests directed to:
`http://<your-domain>/Tunnel/index.php?action=<action_name>`

---

### A. Validate License (`action=validate`)
Queries the license authority to determine the authoritative status, expiration date, and activation counts.

*   **Request Payload (JSON):**
    ```json
    {
      "license_key": "eyJzb2Z0d2FyZV9pZCI6MSwibGljZW5zZV9pZCI6MTIzfQ.signature"
    }
    ```
*   **Success Response (200 OK):**
    ```json
    {
      "success": true,
      "license_id": 123,
      "status": "active",
      "expires_at": "2026-12-31 23:59:59",
      "activation_limit": 5,
      "activation_count": 2,
      "activations": [
        {"domain": "site1.com", "machine_id": "mach-abc-123", "last_active_at": "2026-07-13 12:00:00"},
        {"domain": "site2.com", "machine_id": "mach-xyz-789", "last_active_at": "2026-07-13 12:30:00"}
      ]
    }
    ```

---

### B. Activate License (`action=activate`)
Binds the license to a specific installation domain and machine fingerprint.

*   **Request Payload (JSON):**
    ```json
    {
      "license_key": "eyJzb2Z0d2FyZV9pZCI6MSwibGljZW5zZV9pZCI6MTIzfQ.signature",
      "domain": "myclientwebsite.com",
      "machine_id": "unique-machine-hash-88192"
    }
    ```
*   **Success Response (200 OK):**
    ```json
    {
      "success": true,
      "message": "Activation successful.",
      "license_id": 123
    }
    ```
*   **Error Response (Limit Reached):**
    ```json
    {
      "success": false,
      "error": "Activation limit reached. Please deactivate an existing installation first."
    }
    ```

---

### C. Deactivate License (`action=deactivate`)
Removes an active installation fingerprint to free up a slot.

*   **Request Payload (JSON):**
    ```json
    {
      "license_key": "eyJzb2Z0d2FyZV9pZCI6MSwibGljZW5zZV9pZCI6MTIzfQ.signature",
      "domain": "myclientwebsite.com",
      "machine_id": "unique-machine-hash-88192"
    }
    ```
*   **Success Response (200 OK):**
    ```json
    {
      "success": true,
      "message": "Deactivation successful."
    }
    ```

---

### D. Heartbeat Check-In (`action=heartbeat`)
Updates the last-seen status of a registered activation slot to keep track of active installs.

*   **Request Payload (JSON):**
    ```json
    {
      "license_key": "eyJzb2Z0d2FyZV9pZCI6MSwibGljZW5zZV9pZCI6MTIzfQ.signature",
      "domain": "myclientwebsite.com",
      "machine_id": "unique-machine-hash-88192"
    }
    ```
*   **Success Response (200 OK):**
    ```json
    {
      "success": true,
      "status": "active",
      "expires_at": "2026-12-31 23:59:59",
      "message": "Heartbeat recorded."
    }
    ```
